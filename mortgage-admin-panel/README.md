# Панель администратора ипотечного калькулятора

Прототип интерфейса для управления банками-партнерами и ипотечными программами. Приложение позволяет добавлять новые банки, редактировать условия по ипотечным программам (ставки, взносы, сроки), а также управлять видимостью банков в публичном калькуляторе.

## Функциональность

- **Список банков**: Просмотр всех добавленных банков с логотипами и статусами.
- **Фильтрация**: Быстрый поиск по конкретному банку, статусу активности или наличию программ.
- **Редактирование**: Управление общей информацией о банке (название, логотип, описание).
- **Ипотечные программы**: Добавление и настройка программ (Семейная, IT, Стандартная и др.) с указанием ставок, ПВ и сроков.
- **Автоставки**: Возможность переключения программ в режим автоматического обновления ставок (симуляция).
- **Массовые действия**: Переключатель "Применить ко всем проектам".

## Установка и запуск

### Локальное развертывание на компьютере

1. **Клонируйте репозиторий** (если еще не склонирован):
   ```bash
   git clone <repository-url>
   cd mortgage-admin-panel
   ```

2. **Установите зависимости**:
   ```bash
   npm install
   ```
   
   > **Примечание**: Убедитесь, что у вас установлен Node.js версии 16 или выше. Проверить можно командой `node --version`.

3. **Запустите режим разработки**:
   ```bash
   npm run dev
   ```
   
   Приложение будет доступно по адресу `http://localhost:5173` (или другой порт, если 5173 занят).

4. **Сборка для продакшена**:
   ```bash
   npm run build
   ```
   
   Собранные файлы будут находиться в папке `dist/`.

5. **Предпросмотр продакшен-сборки**:
   ```bash
   npm run preview
   ```
   
   Это запустит локальный сервер для проверки собранного приложения.

### Развертывание на GitHub Pages

#### Вариант 1: Автоматический деплой через GitHub Actions (рекомендуется)

1. **Создайте репозиторий на GitHub** (если еще не создан):
   - Перейдите на [GitHub](https://github.com)
   - Создайте новый репозиторий (например, `mortgage-admin-panel`)

2. **Настройте репозиторий**:
   ```bash
   git init
   git add .
   git commit -m "Initial commit"
   git branch -M main
   git remote add origin https://github.com/ВАШ_USERNAME/mortgage-admin-panel.git
   git push -u origin main
   ```

3. **Включите GitHub Pages**:
   - Перейдите в Settings → Pages вашего репозитория
   - В разделе "Source" выберите "GitHub Actions"
   - Сохраните изменения

4. **GitHub Actions автоматически задеплоит приложение**:
   - При каждом push в ветку `main` будет запускаться автоматический деплой
   - После успешного деплоя приложение будет доступно по адресу:
     `https://ВАШ_USERNAME.github.io/mortgage-admin-panel/`

#### Вариант 2: Ручной деплой

1. **Соберите проект**:
   ```bash
   npm run build
   ```

2. **Настройте репозиторий для GitHub Pages**:
   ```bash
   # Перейдите в папку dist
   cd dist
   
   # Инициализируйте git (если еще не инициализирован)
   git init
   git add .
   git commit -m "Deploy to GitHub Pages"
   
   # Создайте ветку gh-pages
   git branch -M gh-pages
   
   # Добавьте remote (замените на ваш репозиторий)
   git remote add origin https://github.com/ВАШ_USERNAME/mortgage-admin-panel.git
   
   # Запушьте в ветку gh-pages
   git push -u origin gh-pages
   ```

3. **Включите GitHub Pages**:
   - Перейдите в Settings → Pages вашего репозитория
   - В разделе "Source" выберите ветку `gh-pages` и папку `/ (root)`
   - Сохраните изменения

4. **Приложение будет доступно** по адресу:
   `https://ВАШ_USERNAME.github.io/mortgage-admin-panel/`

   > **Примечание**: После первого деплоя может потребоваться несколько минут для активации сайта.

#### Обновление деплоя

- **При автоматическом деплое**: Просто сделайте `git push` в ветку `main`
- **При ручном деплое**: Повторите шаги 1-2 из варианта 2
